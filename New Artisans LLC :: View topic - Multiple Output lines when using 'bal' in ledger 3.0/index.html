Ever since upgrading to ledger 3.0, I have been receiving multiple lines of output for accounts when using the 'bal' command. Here is an example, of checking the balance of my two checking accounts. The first is with ledger 2.5, the second with the CVS source version of ledger 3.


Code:

ktneely@piglet:/tmp/ledger_2_5$ ./ledger bal Checking
            $ 293.13  Assets:Bank1:Checking
           $ -210.58  Assets:Bank2:Checking
--------------------
             $ 82.55

ktneely@piglet:/tmp/ledger3$ ./ledger bal Checking
            $ 794.89  Assets:Bank1:Checking
           $ -501.76  Assets:Bank1:Checking
            $ -18.58  Assets:Bank2:Checking
           $ -192.00  Assets:Bank2:Checking
--------------------
             $ 82.55


As you can see, they add up, but I am unsure as to what is causing the separate entries. What should I look for to eliminate the second entries. I cannot find any specific entry with those numbers, so it is definitely multiple entries in my ledger.dat.

Also, nothing looks funny when I run the register using 'reg'. It all adds up just fine.

Posted: 16 Mar 2007 11:52 am    Post subject: SOLVED
I believe I have figured out what is causing this problem. ledger 3 CVS appears to differentiate between account names with spaces after them and those that do not have any spaces, even if the rest of the account name is identical.

I was able to "move" the money around between the identical accounts by placing or removing a space after an entry. I think I see this because I let ledger reconcile the negative amount for the transaction using its default behavior of making an equal negative entry.
